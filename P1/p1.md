# Essay Singkat:

1. Jelaskan minimal 3 perbedaan antara frontend dan backend pengembangan web!
2. Gambarkan dan jelaskan alur request-response HTTP dari browser hingga database!
3. Sebutkan 3 entitas data utama dalam Sistem Perpustakaan yang akan kita bangun, beserta minimal 3 atributnya masing-masing!
# JAWABAN
1. * Lokasi Eksekusi: Frontend berjalan di sisi klien (browser pengguna), sedangkan backend berjalan di sisi server (seperti pada server lokal XAMPP atau Laragon).
    * Fokus Utama: Frontend berfokus pada desain antarmuka, pengalaman pengguna (UX), dan estetika. Backend berfokus pada logika bisnis, keamanan, dan integrasi database.
    * Bahasa Pemrograman: Frontend menggunakan HTML, CSS, dan JavaScript. Backend menggunakan bahasa seperti PHP (versi 8.x ke atas), Python, atau Java.

2. Alur komunikasi data di internet mengikuti siklus yang disebut Request-Response:
    1. Request (Permintaan): Pengguna memasukkan alamat URL di browser. Browser mengirimkan pesan HTTP Request ke web server.
    2. Server Processing: Web server (seperti Apache di Laragon) menerima permintaan tersebut. Jika kode bersifat dinamis, PHP akan memproses logika program.
    3. Database Access: Jika dibutuhkan data (misal: daftar buku), PHP akan meminta data ke database MySQL melalui phpMyAdmin atau koneksi langsung.
    4. Response (Tanggapan): Server membungkus hasil proses menjadi file HTML atau data JSON, lalu mengirimkannya kembali sebagai HTTP Response ke browser untuk ditampilkan kepada pengguna.


            1. User mengetik URL di browser
                    ↓
            2. Browser mengirim HTTP Request ke Web Server (Apache)
                    ↓
            3. Apache meneruskan ke PHP Engine
                    ↓
            4. PHP mengeksekusi kode dan query ke Database MySQL
                    ↓
            5. MySQL mengembalikan hasil query ke PHP
                    ↓
            6. PHP membangun HTML dari data yang diterima
                    ↓
            7. Apache mengirim HTTP Response (HTML) ke browser
                    ↓
            8. Browser merender dan menampilkan halaman ke pengguna

3. - Buku: Objek fisik yang dipinjam. Atribut: id_buku (Primary Key), judul_buku, pengarang.

    - Anggota: Orang yang terdaftar di perpustakaan. Atribut: id_anggota, nama_lengkap, alamat_email.

    - Transaksi: Catatan transaksi antara anggota dan buku. Atribut: id_pinjam, tanggal_pinjam, status_kembali.
